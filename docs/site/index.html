okeytokey

Design tokens, decided. W3C DTCG-native, local-first.

An enterprise-grade design token platform: a headless core engine, a web studio, Git sync with a connection doctor, an export pipeline, and a Figma plugin. Every token can carry the decision behind it — guidelines, lifecycle, rationale — under the com.okeytokey extension namespace, and every file remains a valid DTCG document.

Why okeytokey

  • Lossless round-trips — key order and unknown fields survive, always.
  • Rename-with-refactor across every set, alias, math expression, and composite. Free.
  • A lint engine: broken references, cycles, WCAG + APCA contrast, orphans, deprecated usage.
  • Semantic diffs with transitive impact analysis — “this change affects 47 tokens”.
  • Sync that explains itself: structured traces, a connection doctor, dry-run pushes, token-level merges.
  • OKLCH-native color with sRGB gamut warnings.

Documentation

This site is a stub. For now, the source of truth is the repository: per-package READMEs document every public API, and docs/adr/ records the architecture decisions (resolution semantics, extension namespace, undo model, sync/merge strategy).

Getting started

pnpm install && pnpm build && pnpm --filter @okeytokey/studio dev — then open http://localhost:5173 and pick a starting point in the onboarding wizard. Build artifacts in CI with pnpm okeytokey build okeytokey.config.json.
